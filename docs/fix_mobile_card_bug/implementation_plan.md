# 実装計画：モバイルでのカード消失バグおよびクリック領域の改善

## 概要
- `Card.tsx` のコンポーネントにおけるクラス操作ロジックを変更する。
- `z-index` （`z-40` クラス）の動的な付与を `className` プロパティではなく `useEffect` で行う。
- ヘルプボタンのクリック領域をスタイル変更で拡大する。

## タスクリスト

### 1. `app/components/Card.tsx` の修正
- [x] コンポーネント内の `useRef` を使用してルートの `div` 要素（`.dashboard-card`）への参照を取得する。
- [x] `showInfo` ステートの変更時に `dashboard-card` に `z-40` を追加/削除する `useEffect` を実装する。
- [x] 親 `div` の `className` 文字列生成から `${showInfo ? 'z-40' : ''}` を削除し、再レンダリング時に `className` が変更されないようにする（`card-visible` の保持）。
- [x] ヘルプボタン (`<button>`) に `p-2` と `-m-2` を追加してクリック領域を拡大する。

### 2. 検証
- [ ] モバイルシミュレーションで "?" マークを押してもカードが表示されたままであることを確認する。
- [ ] クリック領域が適切に拡大されていることを確認する。
